# CPS-VIDA Website Agent Instructions

## Objective
Build a polished, academically credible redesign for the USC CPS-VIDA Lab website.

## Technology
- Next.js with App Router.
- TypeScript.
- Tailwind CSS.
- Static export only.
- All deployable files must be generated by `npm run build` into `out/`.

## Static-hosting constraints
- Configure `output: "export"`.
- Configure `trailingSlash: true`.
- Configure Next Image for static export with unoptimized images.
- Do not create API routes, route handlers, server actions, middleware, authentication, databases, CMS integrations, or server-side rendering.
- The site must work on GitHub Pages.
- Support both:
  1. GitHub Pages preview at `/vida-lab-redesign/`
  2. Final deployment at a root custom domain such as `cps-vida.github.io`

## Content rules
- Do not fabricate people, titles, academic affiliations, research claims, awards, funding, publications, news, contact information, or recruiting information.
- Keep all editable content in clearly organized TypeScript files under `src/content/`.
- Use clearly marked placeholder data where approved information is unavailable.
- Existing public website content may only be used as provisional starter material and must remain easy to replace.

## Design direction
- Use Stanford IRIS Lab only as an information-architecture reference, not as a visual or code template.
- Aim for a restrained, modern academic-research identity.
- Avoid startup aesthetics, gradients used as decoration, carousels, excessive animation, stock photos, AI-generated people, and video backgrounds.
- Use an abstract technical SVG/CSS visual where a hero image would otherwise be needed.
- Prioritize legibility, spacing, responsiveness, and accessibility.

## Required pages
- Home
- Research
- People
- Publications
- Contact

## Required components
- Responsive navigation
- Footer
- Research-area cards
- Reusable person cards
- Reusable publication entries
- News list
- Mobile navigation
- Accessible buttons, links, image alt text, and semantic headings

## Verification
Before completing work:
1. Run linting.
2. Run a production build.
3. Confirm `out/` is generated.
4. Confirm all internal links work.
5. Report placeholders that need approved content.
